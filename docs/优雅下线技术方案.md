# 优雅下线技术方案

## 一、概述

基于需求文档确定的范围，本期实现以下功能：

1. **Provider 端**：
   - 8步优雅下线流程
   - 主动通知（获取长连接列表、发送通知、重试机制、超时控制）
   - 被动通知（响应携带Closing标记）
2. **Consumer 端**：
   - 被动感知（检测响应中的Closing标记）
   - 主动感知（接收长连接关闭通知）
   - 移除不可用实例

### 2.2 Provider Filter 扩展

```go
// filter/graceful_shutdown/provider_filter.go

const (
    GracefulShutdownClosingKey = "closing"
)

type providerGracefulShutdownFilter struct {
    shutdownConfig *global.ShutdownConfig
}

func (f *providerGracefulShutdownFilter) Invoke(ctx context.Context, invoker base.Invoker, invocation base.Invocation) result.Result {
    if f.rejectNewRequest() {
        return f.getRejectedExecutionHandler(invoker, invocation)
    }
    f.shutdownConfig.ProviderActiveCount.Inc()
    f.shutdownConfig.ProviderLastReceivedRequestTime.Store(time.Now())
    return invoker.Invoke(ctx, invocation)
}

func (f *providerGracefulShutdownFilter) OnResponse(ctx context.Context, result result.Result, invoker base.Invoker, invocation base.Invocation) result.Result {
    f.shutdownConfig.ProviderActiveCount.Dec()

    // 在 Closing 状态下，响应携带 Closing 标记
    if f.isClosing() {
        result.AddAttachment(constant.GracefulShutdownClosingKey, "true")
    }

    return result
}

func (f *providerGracefulShutdownFilter) isClosing() bool {
    return f.shutdownConfig.Closing.Load()
}
```

### 2.3 Consumer Filter 扩展

```go
// filter/graceful_shutdown/consumer_filter.go

const (
    GracefulShutdownClosingKey = "closing"
)

type consumerGracefulShutdownFilter struct {
    shutdownConfig *global.ShutdownConfig
    closingInvokers sync.Map // map[string]time.Time (url key -> expire time)
}

func (f *consumerGracefulShutdownFilter) Invoke(ctx context.Context, invoker base.Invoker, invocation base.Invocation) result.Result {
    // 检查 Invoker 是否正在关闭
    if f.isClosingInvoker(invoker) {
        return &result.RPCResult{Err: errors.New("provider is closing")}
    }
    f.shutdownConfig.ConsumerActiveCount.Inc()
    return invoker.Invoke(ctx, invocation)
}

func (f *consumerGracefulShutdownFilter) OnResponse(ctx context.Context, result result.Result, invoker base.Invoker, invocation base.Invocation) result.Result {
    f.shutdownConfig.ConsumerActiveCount.Dec()

    // 检测响应中的 Closing 标记
    if f.isClosingResponse(result) {
        f.markClosingInvoker(invoker)
    }

    return result
}

// isClosingInvoker 检查 Invoker 是否在 closing 列表中
func (f *consumerGracefulShutdownFilter) isClosingInvoker(invoker base.Invoker) bool {
    key := invoker.GetURL().String()
    if expireTime, ok := f.closingInvokers.Load(key); ok {
        if time.Now().Before(expireTime.(time.Time)) {
            return true
        }
        f.closingInvokers.Delete(key)
    }
    return false
}

// isClosingResponse 检查响应是否携带 Closing 标记
func (f *consumerGracefulShutdownFilter) isClosingResponse(result result.Result) bool {
    if result != nil && result.Attachments() != nil {
        if v, ok := result.Attachments()[GracefulShutdownClosingKey]; ok {
            if v == "true" {
                return true
            }
        }
    }
    return false
}

// markClosingInvoker 标记 Invoker 为正在关闭
func (f *consumerGracefulShutdownFilter) markClosingInvoker(invoker base.Invoker) {
    key := invoker.GetURL().String()
    expireTime := time.Now().Add(f.getClosingInvokerExpireTime())
    f.closingInvokers.Store(key, expireTime)
    logger.Infof("Marked invoker as closing: %s, will expire at %v", key, expireTime)
}

func (f *consumerGracefulShutdownFilter) getClosingInvokerExpireTime() time.Duration {
    if f.shutdownConfig.ClosingInvokerExpireTime > 0 {
        return f.shutdownConfig.ClosingInvokerExpireTime
    }
    return 30 * time.Second
}
```

## 三、流程设计

### 3.1 Provider 端流程

```
收到 SIGTERM 信号
       │
       ▼
┌──────────────────┐
│ 1. 开始优雅下线   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 2. 标记 Closing  │ ◄── shutdown.Closing.Store(true)
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 3. 主动通知      │ ◄── 遍历长连接，发送关闭通知
│ (支持重试+超时)  │     失败有注册中心兜底
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 4. 反注册        │ ◄── 注册中心作为兜底
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 5. 拒绝新请求   │ ◄── RejectRequest.Store(true)
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 6. 等待请求完成 │ ◄── counter + 滑动窗口
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 7. 等待响应完成 │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
```

### 3.3 滑动窗口算法

```go
// graceful_shutdown/shutdown.go

func waitingProviderProcessedTimeout(shutdown *global.ShutdownConfig, timeout time.Duration) {
    deadline := time.Now().Add(timeout)
    offlineWindowTimeout := shutdown.GetOfflineRequestWindowTimeout()

    for time.Now().Before(deadline) &&
        (shutdown.ProviderActiveCount.Load() > 0 ||
         time.Now().Before(shutdown.ProviderLastReceivedRequestTime.Load().Add(offlineWindowTimeout))) {
        // 轮询间隔 10ms
        time.Sleep(10 * time.Millisecond)
        logger.Infof("waiting for provider active invocation count = %d, provider last received request time: %v",
            shutdown.ProviderActiveCount.Load(), shutdown.ProviderLastReceivedRequestTime.Load())
    }
}
```

## 六、测试方案

### 6.1 单元测试

| 测试用例 | 预期结果 |
|----------|----------|
| Provider Filter OnResponse 在 Closing=true 时携带标记 | 响应 Attachment 包含 closing=true |

### 6.2 集成测试

| 测试场景 | 预期结果 |
|----------|----------|
| Provider 优雅下线，Consumer 收到响应 | Consumer 标记该 Provider 为 closing |
| Provider 优雅下线过程中，Consumer 发起新请求 | Consumer 跳过该 Provider |

| 场景 | 预期 |

## 七、错误处理与容错

### 7.1 核心原则

任意步骤失败不影响整体流程继续。

| 步骤 | 失败处理 | 超时处理 |
|------|----------|----------|
| 标记 Closing 状态 | 记录警告，继续 | 不适用 |
| 主动通知 | 记录警告，继续（有兜底） | 强制进入下一步 |
| 反注册 | 记录警告，继续 | 不适用 |
| 拒绝新请求 | 记录警告，继续 | 不适用 |
| 等待请求完成 | - | 强制进入下一步 |
| 等待响应完成 | - | 强制进入下一步 |
| 销毁协议 | 记录警告，继续 | 不适用 |
| 执行回调 | 记录警告，继续 | 不适用 |

### 7.2 主动通知超时处理

```go
func notifyLongConnectionConsumers() error {
    stepTimeout := getStepTimeout()
    done := make(chan struct{})

    go func() {
        // 执行主动通知
        notifyTripleConsumers()
        notifyGrpcConsumers()
        close(done)
    }()

    select {
    case <-done:
        // 通知完成
        return nil
    case <-time.After(stepTimeout):
        // 超时，强制进入下一步
        logger.Warn("Graceful shutdown: notify timeout, continuing...")
        return nil
    }
}
```

## 八、风险与注意事项

1. **Consumer 端缓存过期**：当前设计使用 30 秒过期时间，需要根据实际场景调整
2. **注册中心通知**：仍有注册中心作为兜底，被动通知失败不影响整体流程
3. **向后兼容**：修改不影响现有业务逻辑
